# ACT-LEAMAS-FACTORY-DIGEST-STAGED-STATUS-CLASSIFICATION01-CORRECTION03

## Title

Resolve the three contract / evidence / self-hosting defects the
CORRECTION02 reviewer surfaced:

1. Apply `PathEscape` to the `REVIEW_MAP` section so unusual
   filenames are escaped at every path-rendering boundary.
2. Migrate the evidence hash scope from
   `normalized_digest_v2_sections` to v3 and contract-test the
   new value.
3. Provide a fresh self-hosting proof built from the final HEAD
   with the producer's git commit recorded in the digest header;
   produce focused fixture proofs for the `T  linked.go`,
   `C  source.go -> copy.go`, and newline-bearing-path scenarios.

## Status

Implemented.

## Context

The corrector 02 round introduced the v3 contract and the
raw-paths-in-semantic-models refactor, but a third reviewer pass
surfaced three remaining defects:

* `RenderReviewMap` writes paths directly into the rendered
  REVIEW_MAP body without escaping them. A path containing an
  embedded newline would split a bullet line across two visual
  lines.
* The evidence-hash scope `normalized_digest_v2_sections` was
  never updated. A v3 digest that still reports the v2 scope is
  misleading even when the hashed material includes the new
  fields.
* The self-hosting proof supplied with the corrector 02 was
  generated by a binary built before the corrector 02 code
  landed; the digest's `LEAMAS_COMMIT` reported the previous
  commit, and the v3 stats keys were not present in the output.

## Goal

Each of the three closure blockers above is fixed without
regressing the corrector 02 work.

## Hard constraints

The parent ACT and CORRECTION01 / CORRECTION02 hard constraints
remain in force.

## Approach

1. Update `RenderReviewMap` to apply `PathEscape` to each bullet
   path. Add `TestReviewMap_NewlinePath` that commits a path
   with embedded newlines, asserts the REVIEW_MAP body does not
   contain the raw path, and asserts the escaped form is
   present.
2. Migrate `EvidenceHashScope` to
   `normalized_digest_v3_sections`. Update the unit and
   integration tests accordingly. Document the v3 scope in the
   v3 contract section of `docs/factory/digest-contract.md`.
3. After the corrector 03 commit, run
   `make build` (or the equivalent `go build` invocation),
   confirm `LEAMAS_COMMIT` matches `git rev-parse HEAD`, and
   produce:
   * a `HEAD~1..HEAD` (or full correction-range) digest from
     the final HEAD that demonstrates v3 in the header and the
     new v3 stats keys in `CHANGESET_STATS`;
   * a focused staged-mode fixture proof with a regular file
     staged-added and a symlink in the same commit, asserting
     `T  linked.go`;
   * a focused range-mode fixture proof with `git add -N` and a
     `cp` (or `--find-copies-harder` use) that produces a
     `C  source.go -> copy.go` rendered line;
   * a focused staged-mode fixture proof with a path containing
     an embedded newline, asserting the manifest line is escaped
     and the raw path is not present in any rendered section.
