and implementation guidance for the [digest contract](./digest-contract.md).

### Simple manifest

```

```

```
C  internal/factory/digest/template.go -> internal/factory/digest/copy.go

```

### With type change (v3)

```

```

## v2 to v3 Migration

### What changed

v3 introduced:

1. **New status codes**: `T` (type change), `R<N>` (renamed with score dropped),
   `C<N>` (copied with score dropped), `X` (unknown), `B` (pairing broken)
2. **New CHANGESET_STATS keys**: `type_changed_files`, `unknown_files`,
   `broken_pair_files`
3. **Path escaping**: Non-printable characters in paths are escaped
4. **Full rename/copy paths**: Both old and new paths for R/C entries

### Compatibility

- v2 consumers reading v3 output will see unrecognized status codes
- v2 consumers can ignore unknown `CHANGESET_STATS` keys
- The `CHANGESET_STATS` key order changed; v2 consumers reading the first
  N keys will still parse correctly

### Migration guidance

1. Update parsers to handle the new status codes gracefully
2. Update stats aggregation to compute the three new keys
3. Update manifest rendering to include both paths for R/C entries
4. Update path escaping to handle the full character set

```

```

### Binary files

Files containing null bytes (`\x00`).

## Path Escaping Examples (v3)

Characters that get escaped:

| Character | Escape |
|-----------|--------|
| Tab | `\t` |
| Newline | `\n` |
| Carriage return | `\r` |
| Backslash | `\\` |
| Control bytes | `\xXX` |

Example:
```
## CHANGESET_MANIFEST
M  path/with\tembedded\ttab.go
M  path/with\nnewline.go
M  normal/path.go
```
